# <a name="technical-support-for-on-premises-or-cloud-products"></a>Soporte técnico para productos locales o en la nube

**Roles adecuados:**
- Administrador global asociado de MPN o administrador de asociados de MPN para activar sus ventajas de soporte técnico.
- Cualquier cuenta de usuario asociada a MPN para ver el identificador de acceso y el identificador de contrato después de la activación.
- Agente de administración para crear soporte técnico en nombre del cliente para productos en la nube.
- Cualquier cuenta AAD o MSA para crear solicitudes de soporte técnico locales en nombre de su cliente o de su propia organización.

## <a name="overview"></a>Información general

La [página Ventajas técnicas](https://partner.microsoft.com/dashboard/mpn/membership/benefits/technical) muestra información sobre dos ventajas independientes:

1. **Incidentes de soporte técnico del producto: consulte a continuación**
2. Servicios técnicos de preventa e implementación (TPD): [más información](technical-benefits.md)

Este artículo está para los miembros de Microsoft Partner Network (MPN) que necesitan saber cómo activar sus ventajas técnicas, ver el identificador de acceso y el identificador de contrato, saber qué productos y versiones se admiten y cómo iniciar el proceso para crear un incidente de soporte técnico o una incidencia mediante las ventajas de asociados de MPN.

### <a name="steps"></a>Pasos

1. Active sus ventajas técnicas (requiere administrador global o administrador de asociados de MPN). Después de la activación, todas las cuentas de usuario de MPN pueden ver el identificador de acceso y el identificador de contrato para asociar las ventajas a su cuenta.

2. Compruebe los derechos de soporte técnico y los productos admitidos para ver si las ventajas de MPN se pueden usar para el producto y la versión previstos.

3. Soporte técnico de productos de Microsoft (para productos locales recientes): use Soporte técnico para empresas para el soporte técnico local de productos

4. Soporte técnico Signature Cloud (para productos en la nube): los usuarios con el rol de agente de administración **seleccionan Clientes y** administración de servicios para crear una solicitud. Consulte los detalles siguientes sobre cada paso:

## <a name="view-and-activate-your-technical-support-benefits"></a>Visualización y activación de las ventajas de soporte técnico

Cómo ver o activar sus ventajas técnicas en Centro de partners:

1. Inicie sesión con la cuenta asociada a MPN en el [panel](https://partner.microsoft.com/dashboard)del Centro de partners.

   > [!NOTE]
   > Si su cuenta solo está asociada a CSP y no está asociada a MPN, verá un mensaje de error "No está autorizado para esta acción". Debe tener una cuenta asociada a MPN.

2. Expanda **Ventajas en** el menú de navegación izquierdo y seleccione Ventajas [**técnicas.**](https://partner.microsoft.com/dashboard/mpn/membership/benefits/technical) En la página Ventajas técnicas se mostrarán las ofertas de suscripción gold, silver o microsoft Action Pack las organizaciones activas.

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-1.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-1.png" alt-text="A continuación, expanda Ventajas." :::

   > [!NOTE]
   > Si no tiene ninguna oferta de pertenencia activa, verá el mensaje "Parece que aún no tiene ninguna beneficios". Si su empresa tiene ofertas de suscripción activas, pero no las puede ver, ha iniciado sesión con la cuenta incorrecta. Para comprobar la cuenta, seleccione el icono **Cuenta** en la parte superior derecha de la página y seleccione Iniciar sesión **con otra** cuenta (si es necesario).

3. Para usar soporte técnico de productos de Microsoft o soporte técnico para productos de firma, debe activar la ventaja. Para activar la ventaja, seleccione el **botón** Activar. La cuenta de usuario usada para la activación requiere los roles Administrador global o Administrador de asociados de MPN.

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-2.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-2.png" alt-text="Ventajas técnicas seleccione Activar." :::

4. El identificador de acceso y el identificador de contrato se mostrarán a todos los usuarios después de la activación correcta. Puede usar soporte técnico de productos de Microsoft y soporte técnico para productos de firma (miembros Gold y Silver) cuando se muestran el identificador de acceso y el identificador de contrato.

   Si el botón Activar está en gris, la cuenta no tiene el rol de seguridad Administrador global o Administrador de asociados de MPN. Use  [Administración de usuarios](https://partner.microsoft.com/dashboard/account/v3/usermanagement#users) y filtre por Administrador global o Administrador de asociados de MPN para buscar un usuario para activar sus ventajas.

   :::image type="content" source="images/mpn-benefits-technical-support/technical-benefits-3.png" lightbox="images/mpn-benefits-technical-support/technical-benefits-3.png" alt-text="Las ventajas técnicas activadas muestran el identificador de acceso y el identificador de contrato." :::

   > [!NOTE]
   > Necesitará el identificador de acceso y el identificador de contrato la primera vez que use soporte técnico de productos de Microsoft o soporte técnico de firma para cada cuenta. Una vez asociadas las ventajas a su cuenta, no es necesario volver a usar el identificador de acceso y el identificador de contrato.  

## <a name="technical-support-entitlement-and-supported-products"></a>Derechos de soporte técnico y productos admitidos

El soporte técnico proporciona solución de problemas para un problema específico, un mensaje de error o una funcionalidad que no funciona según lo previsto. MpN Product Support and Soporte técnico Signature Cloud support can be used for specific Microsoft products and versions (No se admiten todos los productos y versiones).

> [!NOTE]
> Las ventajas de soporte técnico deben activarse antes de poder usarlas.

Como parte de la suscripción Microsoft Partner Network, los asociados son aptos para el siguiente soporte técnico

|  Soporte técnico |  Action Pack |  Competencia Silver | Competencia Gold |
|:------------------:|:------------:|:------------------:|:---------------:|
| Soporte técnico de productos de Microsoft<br>Para el entorno local reciente <br>solo productos | 10 incidentes | 15 incidentes | 20 incidentes |
| Soporte técnico Signature Cloud<br>Solo para productos en la nube | | Competencias en la nube:<br>Sin límite<br><br>Otras competencias:<br>5 incidentes | Competencias en la nube:<br>Sin límite<br>          <br>Otras competencias:<br>10 incidentes   | 

Las competencias en la nube incluyen: Competencia de seguridad, Cloud Business Applications, Cloud Platform, Cloud Productivity, Enterprise Mobility Management, Soluciones en la nube pequeñas y medianas.

## <a name="microsoft-product-support-on-premises"></a>Soporte técnico de productos de Microsoft (local)

Puede usar soporte **técnico de Microsoft** para soporte técnico en productos locales recientes. Los productos locales recientes incluyen solo el producto actual (N) y la versión anterior del producto (N-1). La versión anterior del producto solo se admite si está dentro del [soporte estándar](/lifecycle/policies/fixed); según la directiva [de soporte técnico N-1.](https://aka.ms/n-1) En el caso de los productos y versiones no compatibles, use la opción de soporte técnico de pago correspondiente o compre un plan de soporte técnico para partners.

Necesitará el identificador de acceso y el identificador de contrato la primera vez que use la ventaja. Las ventajas están asociadas a su cuenta de usuario hasta que expiran o no quedan más incidentes de soporte técnico.

Cualquier cuenta de usuario (AAD o MSA) puede usar soporte técnico para productos locales si tienen el identificador de acceso y el identificador de contrato.

Seleccione [Soporte técnico para](https://support.serviceshub.microsoft.com/supportforbusiness/create) empresas para crear una solicitud de soporte técnico para un producto local.

### <a name="how-many-partner-support-incidents-are-remaining"></a>Cuántos incidentes de soporte técnico de asociados quedan

Solo puede obtener esta información en el portal de soporte técnico local de Soporte técnico para empresas. Desafortunadamente, los portales en la nube (por ejemplo, Azure, LCS y otros) no muestran esta información.

Debe usar Soporte técnico para empresas e iniciar el proceso para crear un incidente de soporte técnico local. No creamos una solicitud de soporte técnico porque no completamos el proceso. La información que necesitamos se muestra en el paso 2.

1. Abra [Soporte técnico para empresas,](https://support.serviceshub.microsoft.com/supportforbusiness/create) seleccione Windows 10 y escriba información ficticia.

2. Si el contrato de soporte técnico ya está asociado a su cuenta, debería ver los incidentes de soporte técnico mostrados. Para obtener soporte técnico local, debería ver que se muestra el contrato de soporte técnico para partners (a menos que ya no sea válido), incluida la fecha de inicio, la fecha de finalización y los incidentes restantes.

Si necesita asociar el contrato de soporte técnico, expanda "Do you need  to purchase or add more support plans" (¿Necesita comprar o agregar más planes de soporte técnico?) y seleccione el botón Agregar contrato para escribir el identificador de acceso y el identificador del contrato. Expanda **¿Desea ver planes de** soporte técnico no disponibles? sección para ver la información de la firma de asociado. Solo verá el soporte técnico para partners que se muestra aquí si los contratos han expirado, no queda ningún incidente o ha seleccionado un producto local antiguo (consulte aka.ms/n-1 para productos de soporte técnico). [](https://aka.ms/n-1)

Para obtener información más detallada, consulte [¿Cuántos incidentes de soporte técnico de asociados quedan?](https://support.microsoft.com/topic/how-many-partner-support-incidents-are-remaining-e8220db5-14f6-93db-4b4a-35766e11b3c3).

## <a name="signature-cloud-support-cloud-only"></a>Soporte técnico Signature Cloud (solo en la nube)

Los asociados de las competencias Gold o Silver pueden usar **Soporte técnico Signature Cloud** soporte técnico en productos en la nube. Action Pack asociados no son aptos para Soporte técnico Signature Cloud a menos que obtengan una competencia.  

Necesitará el identificador de acceso y el identificador de contrato la primera vez que use la ventaja para Azure, Dynamics 365 y otros productos en la nube (excepción Microsoft 365). Las ventajas están asociadas a su cuenta de usuario hasta que expiran o no quedan más incidentes de soporte técnico.

> [!NOTE]  
> Cuando se usa mediante el Azure Portal. Si el contrato de soporte técnico ya está asociado a la cuenta, verá un mensaje de error que indica que el identificador de acceso ya está asociado..." al agregar el identificador de acceso y el identificador de contrato. vea a continuación

Los asociados de CSP que crean solicitudes de Azure pueden usar el plan de soporte técnico "Proveedor de soluciones en la nube" sin agregar un contrato de soporte técnico.

Los usuarios **con el rol agente de** administración pueden seleccionar [Clientes](https://partner.microsoft.com/commerce/customers/list) en CSP en el menú de navegación izquierdo, seleccionar **Administración** de servicios y **nueva solicitud.**  Al seleccionar la carga de trabajo correspondiente, se abrirá el portal específico de la carga de trabajo, se agregará un contrato de soporte técnico y se creará una solicitud.

### <a name="azure-portal-error-access-id-is-already-associated-"></a>Azure Portal error "El identificador de acceso ya está asociado..."

Cuando use el Azure Portal, use Soporte técnico Signature Cloud seleccionando Ayuda y soporte técnico, Planes de soporte técnico y, a continuación, "Vincular ventajas de soporte técnico" y escriba el identificador de acceso y el identificador de contrato.

> [!NOTE]
> Si tiene Microsoft Action Pack no obtiene soporte técnico en la nube porque no incluye Soporte técnico Signature Cloud: solo soporte técnico de Microsoft (local).

Es posible que vea el mensaje de error `This Access ID is already associated to an account. Please contact your Technical Account Manager to obtain a new Access ID` . Este mensaje de error no es válido y debe omitirse. Hemos solicitado que el error se deshabilite y se reemplazó por un mensaje de error de diagnóstico útil.

Verá el error anterior si asocia un contrato de soporte técnico a una cuenta más de una vez al `Access ID is already associated …` usar el Azure Portal. Consulte los escenarios siguientes:

1. Si tiene ventajas técnicas gold o silver y ha agregado previamente el contrato para el soporte técnico local, use el Azure Portal y agregue el contrato una segunda vez a la misma cuenta. Solo necesita asociar el contrato de soporte técnico a una cuenta una vez y, por tanto, no es necesario vincular las ventajas de soporte técnico en Azure. Si usa las ventajas de soporte técnico de Link cuando el contrato ya está asociado a la cuenta, verá el error y se debe omitir. Use + New Support Request y cree un incidente de soporte técnico con normalidad y se mostrará el plan de soporte técnico.

2. Si el contrato de soporte técnico ha expirado o el paquete de firma 10 (competencia híbrida Gold) o el paquete de firma 5 (competencia híbrida Silver) no tienen ningún incidente restante, el contrato de soporte técnico se ocultará porque no es válido. Si vincula las ventajas de soporte técnico de nuevo, verá el error porque el contrato ya está asociado. Puede comprar un nuevo plan de soporte técnico para obtener incidentes de soporte técnico de Azure.

Consulte planes de soporte técnico para partners a continuación para obtener ofertas de soporte técnico de pago para asociados.

## <a name="partner-support-plans"></a>Planes de soporte técnico para asociados

Puede obtener soporte técnico de pago de Microsoft que incluye soporte técnico y una variedad de opciones de servicio adicionales.

Puede comparar [los planes de soporte técnico de asociados](https://partner.microsoft.com/support/partnersupport) o ver lo siguiente para planes específicos de asociados:

- [soporte técnico Premier para asociados:](https://partner.microsoft.com/support/microsoft-services-premier-support) el nivel superior de soporte técnico de Microsoft
- [Soporte técnico avanzado para asociados:](https://partner.microsoft.com/support/advanced-cloud-support) solo productos en la nube
