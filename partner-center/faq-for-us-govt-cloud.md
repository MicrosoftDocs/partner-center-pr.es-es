# <a name="frequently-asked-questions-for-partner-center-for-microsoft-cloud-for-us-government"></a>Preguntas más frecuentes sobre las Centro de partners para Microsoft Cloud for US Government

**Se aplica a**: Centro de partners para Microsoft Cloud for US Government

**Roles apropiados**: administrador global

## <a name="general-questions"></a>Preguntas generales

Use estas preguntas más frecuentes para obtener más información sobre el programa Proveedor de soluciones en la nube (CSP) para Microsoft Cloud for US Government. También puede ponerse en contacto con el administrador de cuentas para obtener más información.

### <a name="what-is-microsoft-cloud-for-us-government"></a>¿Qué es Microsoft Cloud for US Government?

Diseñado para el gobierno de EE. UU. desde el primer momento, Microsoft Cloud for US Government permite a los clientes del sector público de Estados Unidos(desde grandes agencias federales hasta gobiernos de pequeñas localidades) seleccionar entre una variedad de servicios de informática en la nube. Para satisfacer las necesidades específicas de los clientes, Microsoft ha realizado una inversión significativa en centros de datos y se dedica a cumplir los requisitos, los requisitos y las directivas federales y estatales de Estados Unidos.

### <a name="who-is-eligible-for-the-csp-program-for-microsoft-cloud-for-us-government"></a>¿Quién puede participar en el programa CSP de Microsoft Cloud for US Government?

Los asociados de la Estados Unidos que atienden a entidades gubernamentales federales, estatales y locales de Estados Unidos en sus capacidades gubernamentales son aptos para el programa CSP para Microsoft Cloud for US Government. Microsoft tiene un programa de validación estricto para determinar la idoneidad antes de que los asociados puedan acceder a Microsoft Cloud for US Government. Para poder participar, necesitas demostrar que trabajas con los clientes del gobierno. El proceso de registro para el programa CSP en Microsoft Could for US Government se ha aumentado para capturar más datos para realizar la validación. Los tipos de prueba pueden incluir, pero no se limitan a, números de contrato de gobierno y una carta de patrocinio del cliente del gobierno.

### <a name="is-customer-pricing-the-same-as-for-csp-commercial"></a>¿Son los precios de cliente los mismos que para CSP Commercial?

No. Los precios reflejarán los costos adicionales para proporcionar el valor único de Microsoft Cloud for US Government.

### <a name="is-the-partner-discounting-model-the-same"></a>¿Es igual el modelo de descuento para partners?

Sí. El modelo de descuento de asociados para el programa CSP Microsoft Cloud for US Government es el mismo que el modelo de descuento de asociados para CSP Commercial.

### <a name="csp-commercial-is-an-existing-partner-program-how-is-the-csp-program-for-microsoft-government-cloud-different"></a>CSP Commercial es un programa de partners ya existente. ¿En qué se diferencia del programa CSP de Microsoft Government Cloud?

El programa CSP para Microsoft Cloud for US Government es diferente de CSP Commercial en las áreas siguientes:

- Elegibilidad: el programa CSP para Microsoft Cloud for US Government solo está disponible para los asociados de Estados Unidos que atienden entidades gubernamentales locales, estatales y federales de EE. UU. en sus capacidades gubernamentales. Microsoft tiene un estricto programa de validación para determinar quién puede participar antes de que los partners puedan acceder a la nube de Microsoft Government.

- Incorporación: los partners deben incorporarse al programa CSP de Microsoft Cloud for US Government independientemente del programa CSP Commercial.

- Disponibilidad del producto: tanto Azure Government servicios basados en licencias como Office 365, Enterprise Mobility + Security y Dynamics 365 están disponibles en el programa CSP para Microsoft Cloud for US Government.

- Contrato: El contrato de Revendedor de nube de Microsoft (MCRA) se ha actualizado para reflejar las condiciones únicas del gobierno.

- Funcionalidad del Centro de partners: determinadas características del Centro de partners no se habilitarán para el programa CSP de Microsoft Cloud for US Government para cumplir con los requisitos de cumplimiento y de regulación. Los partners pueden usar Azure Active Directory (Azure AD) o la [API de Microsoft Graph](/partner-center/develop/developing-for-partner-center-for-microsoft-national-cloud#partner_center_msftcloudUS) en su lugar para realizar tareas relacionadas con la administración de usuarios y grupos. La característica solicitud de relación para admitir escenarios multicanal y multi partner estará disponible después del inicio cuando los servicios basados en licencias estén disponibles en CSP para Microsoft Cloud for US Government.

### <a name="i-am-not-a-csp-partner-yet-where-can-i-find-more-information-on-how-to-become-a-csp-partner"></a>Todavía no soy asociado de CSP. ¿Dónde puedo encontrar más información sobre cómo convertirme en asociado de CSP?

Para obtener más información acerca de los modelos de inscripción disponibles para los partners, consulta [Proveedor de soluciones en la nube de Microsoft Partner Network](https://partner.microsoft.com/cloud-solution-provider).

### <a name="how-does-licensing-work"></a>¿Cómo funciona la concesión de licencias?

*Contrato de Revendedor de nube de Micosoft (MCRA)*

El MCRA autoriza a partners a revender utilizando CSP. La versión de US Government Cloud de este acuerdo se acepta al incorporarse a Microsoft Cloud for US Government. Esta versión tiene el apéndice de US Government Cloud y el documento A después del contrato estándar de revendedores de la nube.

Existen dos diferencias entre el MCRA para Microsoft Cloud for US Government y el MCRA estándar:

- El anexo de US Government Cloud contiene responsabilidad adicional de partner para realizar transacciones con los servicios de US Government Cloud.

- La prueba A contiene los Contrato de cliente de Microsoft (Contrato de cliente de Microsoft) que deben incorporarse a todos los contratos de servicios en la nube de la Administración Pública de Estados Unidos que se han firmado entre el CSP y su cliente.

*Contrato de cliente de Microsoft (Contrato de cliente de Microsoft) presentado como Prueba A en MCRA*

Los CSP se necesitan para incorporar el documento A en contrato de gobierno creado entre los CSP y los clientes que compren Microsoft Cloud for US Government. El contrato se publica en el Centro de partners y está disponible en inglés.

Contrato de cliente de Microsoft para la nube de la Administración Pública de Estados Unidos es diferente del estándar Contrato de cliente de Microsoft de las dos maneras siguientes:

- Es necesario que se incorpore al contrato de CSP con su cliente.

- Se ha adaptado para cumplir con las leyes y normativas federales, estatales y locales.

*Contrato de distribuidor de Microsoft Cloud (MCDA)*

El Contrato de distribuidor en la nube (MCDA) de Microsoft autoriza a los distribuidores a vender a través de CSP. La versión de US Government Cloud del MCDA tiene el apéndice de US Government Cloud y el documento A después del contrato estándar de Revendedor de nube de Micosoft (MCRA).

Existen dos diferencias entre el MCDA para Microsoft Cloud for US Government y el MCDA estándar:

- El anexo de Microsoft Cloud for US Government contiene responsabilidad adicionales de partner para realizar transacciones con los servicios de US Government Cloud.

- La prueba A contiene Contrato de cliente de Microsoft (Contrato de cliente de Microsoft) que deben incorporarse a todos los contratos de servicios en la nube de la Administración Pública de Estados Unidos firmados entre el CSP y sus clientes.

### <a name="what-are-the-qualifications-to-sell-microsoft-cloud-for-us-government"></a>¿Cuáles son las calificaciones para vender Microsoft Cloud for US Government?

Los criterios de calificación para vender el programa CSP para Microsoft Cloud for US Government son los mismos que los criterios de calificación para CSP Commercial, abiertos tanto a asociados directos como a revendedores indirectos. No es necesario que se le deba volver a convertir en revendedor indirecto para el programa CSP para Microsoft Cloud for US Government si está aprobado como revendedor indirecto para CSP Commercial.

### <a name="i-am-an-existing-partner-what-do-i-need-to-do-to-sell-microsoft-cloud-for-us-government"></a>Soy un partner existente. ¿Qué tengo que hacer para vender Microsoft Cloud for US Government?

Los partners necesitarán incorporarse al programa CSP para Microsoft Cloud for US Government para cumplir con los requisitos de cumplimiento y regulación. (La incorporación de Microsoft Cloud for US Government es independiente de la incorporación para CSP Commercial). Una vez incorporado, puede crear clientes y ofertas de transacción disponibles en Microsoft Cloud for US Government. Si usa LAS API, consulte Desarrollo para Centro de partners [para Microsoft National Cloud.](/partner-center/develop/developing-for-partner-center-for-microsoft-national-cloud#partner_center_msftcloudUS)

### <a name="is-there-any-difference-in-partner-incentives"></a>¿Hay alguna diferencia en los incentivos de partners?

No. Los incentivos de asociados para el programa CSP Microsoft Cloud for US Government son los mismos que los incentivos de asociados para CSP Commercial. Para obtener más información acerca de los incentivos de partner, inicia sesión en el [portal de Microsoft Partner Network](https://partner.microsoft.com/membership/partner-incentives).

### <a name="are-there-any-changes-to-billing"></a>¿Hay cambios en la facturación?

No. Los criterios de facturación y los requisitos del programa CSP para Microsoft Cloud for US Government son los mismos que los criterios de facturación y los requisitos de CSP Commercial. Para obtener más información sobre facturación, revisa los recursos en [Microsoft Partner Network](https://partner.microsoft.com/cloud-solution-provider/resources).

### <a name="for-data-hosted-in-commercial-data-centers-is-it-possible-to-change-to-microsoft-cloud-for-us-government"></a>En el caso de los datos hospedados en centros de datos comerciales, ¿es posible cambiar a Microsoft Cloud for US Government?

Para facilitar y mantener la integridad del aislamiento de Microsoft Cloud for US Government, no hay mecanismos para mover los servicios existentes de un cliente a Microsoft Cloud for US Government. Si un cliente o partner tiene Azure, será responsable de cualquier movimiento de datos a un nuevo entorno dentro de Microsoft Cloud for US Government.

## <a name="questions-about-microsoft-cloud-for-us-government-for-azure"></a>Preguntas sobre Microsoft Cloud for US Government para Azure

### <a name="what-is-microsoft-cloud-for-us-government-for-azure"></a>¿Qué Microsoft Cloud for US Government para Azure?

La nube de Azure Government ofrece una plataforma en la nube creada en base a los principios básicos de seguridad, privacidad y control, cumplimento y transparencia. Las entidades del sector público reciben una instancia de Microsoft Azure aislada físicamente que emplea servicios excepcionales de seguridad y cumplimiento críticos para el Gobierno de los Estados Unidos para todos los sistemas y aplicaciones que se han construidos en su arquitectura. Estos servicios incluyen certificaciones de cumplimiento FedRAMP y DoD, acuerdos de nivel de estado CJIS, la capacidad de emitir acuerdos de asociación comercial HIPAA y la compatibilidad para IRS 1075. Dirigido por personas filtradas de Estados Unidos, Azure Government es compatible con múltiples escenarios múltiples para crear e implementar soluciones locales o en la nube. Las entidades del sector público también pueden aprovechar la escalabilidad instantánea y el tiempo de actividad garantizado de un servicio en la nube de hiperescala.

### <a name="does-the-csp-program-for-microsoft-cloud-for-us-government-offer-all-azure-services"></a>¿El programa CSP de Microsoft Cloud for US Government ofrece todos los servicios de Azure?

La infraestructura como servicio (IaaS), la plataforma fundamental y las ofertas de datos ya están disponibles. La hoja de ruta se actualiza de forma continua y proporcionamos documentación similar a la documentación de CSP Commercial Centro de partners (se requiere autenticación). Puede revisar los servicios de Azure Government disponibles actualmente aquí, pero tenga en cuenta que algunos servicios, como el proceso clásico, las redes, el almacenamiento y los servicios en la nube no están diseñados para CSP y no funcionan para CSP.

### <a name="what-do-i-need-to-do-to-develop-with-the-csp-program-for-microsoft-cloud-for-us-government"></a>¿Qué tengo que hacer para desarrollar con el programa CSP de Microsoft Cloud for US Government?

Para obtener más información sobre el desarrollo con Centro de partners para Microsoft Cloud for US Government, vea [Developing with Centro de partners for Microsoft Government Cloud](/partner-center/develop/developing-for-partner-center-for-microsoft-national-cloud#partner_center_msftcloudUS).

### <a name="are-there-any-differences-in-apis"></a>¿Hay alguna diferencia entre las API?

Los requisitos de integración y API del programa CSP de Microsoft Cloud for US Government son idénticos a los de CSP Commercial. Para obtener más información sobre los puntos de conexión Microsoft Cloud for US Government, [vea Centro de partners REST URL](/partner-center/develop/partner-center-rest-urls).

>[!NOTE]
>Las API DE LA MARCA NO estarán disponibles para el programa CSP para Microsoft Cloud for US Government. Estas API están están obsoleto en CSP y, por tanto, no están disponible para Microsoft Cloud for US Government. Para cumplir con los requisitos de cumplimiento y regulación, las siguientes API no estarán disponibles en el programa CSP de Microsoft Cloud for US Government.
>
> **Administración de cuentas de usuario**
>- Crear cuentas de usuario para un cliente
>- Delete user accounts for a customer (Eliminar cuentas de usuario de un cliente)
>- Update user accounts for a customer (Actualizar cuentas de usuario para un cliente)
>- Get a list of all user accounts for a customer (Obtener una lista de todas las cuentas de usuario para un cliente)
>- Reset user password for a customer (Restablecer la contraseña de usuario de un cliente)
>- Get user roles for a customer (Obtener los roles de usuario de un cliente)
>- Set user roles for a customer (Configurar los roles de usuario de un cliente)
>- View deleted users for a customer (Ver los usuarios eliminados de un cliente)
>- Restore a deleted user for a customer (Restaurar un usuario eliminado para un cliente)
>
> **Administración de licencias**
>- Get a list of available licenses (Obtener una lista de licencias disponibles)
>- Assign licenses to a user (Asignar licencias a un usuario)
>-  Check which licenses are assigned to a user (Comprobar qué licencias se asignan a un usuario)

### <a name="is-there-a-workaround-for-apis-not-available-in-the-csp-program-for-microsoft-cloud-for-us-government"></a>¿Existe una solución alternativa para las API que no están disponibles en el programa CSP de Microsoft Cloud for US Government?

Los asociados pueden usar Azure Portal, Azure AD PowerShell o el Graph API administración de cuentas de usuario de asociados. Para obtener más información técnica, consulte [Desarrollo con Centro de partners para government cloud](https://msdn.microsoft.com/partner-center/partner-center-for-microsoft-us-govt-cloud).
