# <a name="frequently-asked-questions-for-partner-center-for-microsoft-cloud-for-us-government"></a>Preguntas más frecuentes acerca del Centro de partners para Microsoft Cloud for US Government

**Se aplica a**

-  Centro de partners para Microsoft Cloud for US Government

## <a name="frequently-asked-questions"></a>Preguntas frecuentes 

Usa estas preguntas más frecuentes para obtener más información acerca del programa Proveedor de soluciones en la nube (CSP) de Microsoft Cloud for US Government. También puedes ponerte en contacto con el administrador de cuentas para obtener más información. 

### <a name="common"></a>Común

**¿Qué es Microsoft Cloud for US Government?**

Diseñado desde cero para el gobierno de los Estados Unidos, Microsoft Cloud para el gobierno permite que los clientes del sector público de los Estados Unidos (desde grandes agencias federales grandes hasta ayuntamientos de pueblos pequeños) elijan entre una variedad de servicios de informática en la nube. Para dirigirse mejor a las necesidades específicas de los clientes, Microsoft ha realizado una gran inversión en centros de datos y se compromete a cumplir las directivas, exigencias y requisitos estatales y federales de los Estados Unidos. 

**¿Quién es apto para el programa CSP de Microsoft Cloud for US Government?**

Los partners en Estados Unidos que sirven a entidades gubernamentales locales, estatales y federales de los Estados Unidos en sus capacidades gubernamentales pueden participar en el programa CSP de Microsoft Cloud for US Government. Microsoft tiene un estricto programa de validación para determinar quién puede participar antes de que los partners puedan acceder a la nube de Microsoft Government. Para poder participar, necesitas demostrar que trabajas con los clientes del gobierno. El proceso de registro en el programa CSP en Microsoft Cloud for US Government se ha aumentado para capturar datos adicionales para realizar la validación. Los tipos de prueba pueden incluir, pero no se limitan a, números de contrato de gobierno y una carta de patrocinio del cliente del gobierno.

**¿Es cliente precios de las mismas que para CSP comercial?**

No. Los precios reflejarán costes adicionales para proporcionar el valor único de Microsoft Cloud for US Government.

**¿Es el mismo modelo de descuentos de socios?**

Sí. El modelo de descuento para partners del programa CSP de Microsoft Cloud for US Government es el mismo que el modelo de descuento de CSP Commercial.

**CSP comercial es un programa asociado existente. ¿Cómo es el programa CSP para la nube de Microsoft Government diferente?**

El programa CSP de Microsoft Government Cloud es diferente del programa CSP Commercial en las siguientes áreas:

-   Elegibilidad para:  El programa CSP para la nube de Microsoft Government solo está disponible para los asociados en Estados Unidos que sirven a US Federal, estatal, y las entidades gubernamentales locales en sus capacidades gubernamentales. Microsoft tiene un estricto programa de validación para determinar quién puede participar antes de que los partners puedan acceder a la nube de Microsoft Government.

-   Incorporación: Los asociados deben incorporar al programa CSP para Microsoft Cloud for US Government por separado de CSP comercial.

-   Disponibilidad del producto: Azure Government y servicios basados en puestos como Office 365, Enterprise Mobility + Security y Dynamics 365 están disponibles en el programa CSP para la nube de Microsoft Government. 

-   Contraer: El acuerdo de revendedor de Microsoft en la nube (MCRA) se ha actualizado para reflejar los términos únicos.

-   Funcionalidad del centro de partners: Ciertas características del centro de partners no se habilitará para el programa CSP de Microsoft Cloud for US Government para cumplir las normas y los requisitos de cumplimiento. Los partners pueden usar Azure Active Directory (Azure AD) o la [API de Microsoft Graph](https://msdn.microsoft.com/library/partnercenter/mt789013.aspx#partner_center_msftcloudUS) en su lugar para realizar tareas relacionadas con la administración de usuarios y grupos. La característica de solicitud de relación para admitir escenarios multicanal y multipartner estará disponible después del lanzamiento cuando los servicios basados en puesto estén disponibles en CSP de Microsoft Cloud for US Government.

**No soy un asociado de CSP todavía. ¿Dónde puedo encontrar más información sobre cómo convertirse en un asociado de CSP?**

Para obtener más información acerca de los modelos de inscripción disponibles para los partners, consulta [Proveedor de soluciones en la nube de Microsoft Partner Network](https://partner.microsoft.com/cloud-solution-provider).

**¿Cómo funciona licencias?**

*Acuerdo de revendedor de nube de Microsoft (MCRA)*

El MCRA autoriza a partners a revender utilizando CSP. La versión de US Government Cloud de este acuerdo se acepta al incorporarse a Microsoft Cloud for US Government. Esta versión tiene el apéndice de US Government Cloud y el documento A después del contrato estándar de revendedores de la nube.

Existen dos diferencias entre el MCRA para Microsoft Cloud for US Government y el MCRA estándar:

-   El anexo de US Government Cloud contiene responsabilidad adicional de partner para realizar transacciones con los servicios de US Government Cloud.

-   Anexo A contiene el contrato de Microsoft en la nube (contrato de Microsoft en la nube) que se deben incorporar en cada contrato de servicios de nube de US Government celebrado entre su cliente y el CSP.

*En la nube de contrato de Microsoft (acuerdo de nube de Microsoft) presentan como anexo A en MCRA*

Los CSP se necesitan para incorporar el documento A en contrato de gobierno creado entre los CSP y los clientes que compren Microsoft Cloud for US Government. El contrato se publica en el Centro de partners y está disponible en inglés.

Existen dos diferencias entre el acuerdo de Microsoft Cloud para US Government Cloud y el acuerdo estándar de Microsoft Cloud:

-   Necesita estar incorporado en el contrato de CSP con su cliente.

-   Se ha adaptado para cumplir las leyes locales, estatales y federales y sus regulaciones.

*Contrato de distribuidor en la nube de Microsoft (MCDA)*

El contrato de distribuidores de Microsoft Cloud (MCDA) autoriza a los distribuidores a vender a través de CSP. La versión de US Government Cloud del MCDA tiene el apéndice de US Government Cloud y el documento A después del contrato estándar de Revendedor de nube de Micosoft (MCRA).

Existen dos diferencias entre el MCDA para Microsoft Cloud for US Government y el MCDA estándar:

-   El anexo de Microsoft Cloud for US Government contiene responsabilidad adicionales de partner para realizar transacciones con los servicios de US Government Cloud.

-   Anexo A incluyen contratos de nube de Microsoft (acuerdo de Microsoft en la nube) que debe incorporarse en cada contrato de servicios de nube de US Government celebra entre el CSP y sus clientes.

**¿Cuáles son los requisitos para vender Microsoft Cloud para el gobierno estadounidense?**

Los criterios de cualificación para vender el programa CSP de Microsoft Government Cloud son los mismos que para CSP Commercial, abierto para partners y revendedores indirectos. No es necesario ser designado de nuevo para ser un revendedor indirecto del programa CSP de Microsoft Government Cloud si estás aprobado como revendedor indirecto del programa CSP en Commercial.

**Soy un asociado existente. ¿Qué es necesario hacer para vender Microsoft Cloud for US Government?**

Los partners necesitarán incorporarse al programa CSP para Microsoft Cloud for US Government para cumplir con los requisitos de cumplimiento y regulación. (La incorporación a Microsoft Cloud for US Government es independiente de la incorporación a CSP Commercial). Cuando te hayas incorporado, podrás crear clientes y tramitar ofertas disponibles en Microsoft Cloud for US Government. Si estás aprovechando las API, consulta [Desarrollo para el Centro de partners de Microsoft National Cloud](https://msdn.microsoft.com/library/partnercenter/mt789013.aspx#partner_center_msftcloudUS).

**¿Hay alguna diferencia en incentivos para partners?**

No. Los incentivos para los partners del programa CSP de Microsoft Cloud for US Government son los mismos que para CSP Commercial. Para obtener más información acerca de los incentivos de partner, inicia sesión en el [portal de Microsoft Partner Network](https://partner.microsoft.com/membership/partner-incentives).

**¿Existen cambios para la facturación?**

No. Los requisitos y criterios de facturación del programa CSP de Microsoft Cloud for US Government son los mismos que los de CSP Commercial. Para obtener más información sobre facturación, revisa los recursos en [Microsoft Partner Network](https://partner.microsoft.com/cloud-solution-provider/resources).

**¿Para los datos hospedados en centros de datos comerciales, es posible cambiar a Microsoft Cloud para el gobierno de Estados Unidos?**

Para facilitar y mantener la integridad del aislamiento de Microsoft Cloud for US Government, no hay ningún mecanismo para trasladar los servicios existentes de un cliente a Microsoft Cloud for US Government. Si un cliente o partner tiene Azure, será responsable de cualquier movimiento de datos a un nuevo entorno dentro de Microsoft Cloud for US Government.


### <a name="microsoft-cloud-for-us-government-for-azure"></a>Microsoft Cloud for US Government para Azure

**¿Qué es Microsoft Cloud for US Government para Azure?**

La nube de Azure Government ofrece una plataforma en la nube creada en base a los principios básicos de seguridad, privacidad y control, cumplimento y transparencia. Las entidades del sector público reciben una instancia de Microsoft Azure aislada físicamente que emplea servicios excepcionales de seguridad y cumplimiento críticos para el Gobierno de los Estados Unidos para todos los sistemas y aplicaciones que se han construidos en su arquitectura. Estos servicios incluyen certificaciones de cumplimiento de FedRAMP y DoD, acuerdos CJIS a nivel de estado, la capacidad para emitir acuerdos de asociados de negocias de la HIPAA y compatibilidad con IRS 1075. Dirigido por personas filtradas de Estados Unidos, Azure Government es compatible con múltiples escenarios múltiples para crear e implementar soluciones locales o en la nube. También pueden tomar entidades del Sector público aprovechar la escalabilidad instantánea y actividad garantizado de un servicio de nube de gran escala.

**¿El programa CSP de Microsoft Cloud for US Government ofrece todos los servicios de Azure?**

La infraestructura como servicio (IaaS), la plataforma fundamental y las ofertas de datos ya están disponibles. El mapa de ruta se actualiza de forma continuada y se proporcionan documentación similar a la documentación de CSP comercial en el centro de partners (se requiere autenticación). Puede revisar los servicios de Azure Government que están disponibles aquí, pero ten en cuenta que algunos servicios, como el cálculo básico, redes, almacenamiento y servicios en la nube no están diseñados para CSP y, por lo tanto, no funcionan.

**¿Qué es necesario hacer para desarrollar con el programa CSP para Microsoft Cloud para el gobierno de Estados Unidos?**

Para obtener más información sobre cómo desarrollar con el Centro de partners de Microsoft Cloud for US Government consulta [Desarrollar con el Centro de partners de Microsoft Government Cloud](https://msdn.microsoft.com/library/partnercenter/mt789013.aspx#partner_center_msftcloudUS).

**¿Existen las diferencias en las API?**

Los requisitos de integración y API del programa CSP de Microsoft Cloud for US Government son idénticos a los de CSP Commercial. Para obtener más información sobre puntos de conexión en Microsoft Cloud for US Government, consulta [Partner Center REST URLs](https://msdn.microsoft.com/library/partnercenter/mt490977.aspx) (URL de REST del Centro de partners).

>[!NOTE]
>Ten en cuenta que las API de CREST no estarán disponibles para el programa CSP de Microsoft Cloud for US Government. Estas API están están obsoleto en CSP y, por tanto, no están disponible para Microsoft Cloud for US Government. Para cumplir con los requisitos de cumplimiento y regulación, las siguientes API no estarán disponibles en el programa CSP de Microsoft Cloud for US Government.

**Administración de cuentas de usuario** 

-   Crear cuentas de usuario para un cliente

-   Delete user accounts for a customer (Eliminar cuentas de usuario de un cliente)

-   Update user accounts for a customer (Actualizar cuentas de usuario para un cliente)

-   Get a list of all user accounts for a customer (Obtener una lista de todas las cuentas de usuario para un cliente)   

-   Reset user password for a customer (Restablecer la contraseña de usuario de un cliente)   

-   Get user roles for a customer (Obtener los roles de usuario de un cliente)    

-   Set user roles for a customer (Configurar los roles de usuario de un cliente)    

-   View deleted users for a customer (Ver los usuarios eliminados de un cliente)    

-   Restore a deleted user for a customer (Restaurar un usuario eliminado para un cliente)    

**Administración de licencias**

-   Get a list of available licenses (Obtener una lista de licencias disponibles)

-   Assign licenses to a user (Asignar licencias a un usuario)

-   Check which licenses are assigned to a user (Comprobar qué licencias se asignan a un usuario)

**¿Existe una solución alternativa para las API no está disponible en el programa CSP para Microsoft Cloud para el gobierno estadounidense?**

Los partners pueden usar el Portal de Azure, Azure AD PowerShell o la API de Microsoft Graph para la administración de cuentas de usuario de partner. Para obtener información técnica adicional, consulta [Desarrollar con el Centro de partners para Government Cloud](https://msdn.microsoft.com/partner-center/partner-center-for-microsoft-us-govt-cloud).
