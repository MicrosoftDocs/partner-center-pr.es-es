## <a name="changes-coming-to-partner-center-apis"></a><a name="9"></a>Próximos cambios en las API del Centro de partners

### <a name="categories"></a>Categories

- Fecha: 28-07-2020
- Funcionalidades

## <a name="summary"></a>Resumen

El equipo del Centro de partners ha evaluado las API expuestas del Centro de partners. Vemos que algunas de estas API están obsoletas y ya no se admiten. Se retirarán el 31 de agosto de 2020.

## <a name="details"></a>Detalles

El equipo del Centro de partners de CSP ha identificado algunas API cuya retirada se está programando para el 31 de agosto de 2020. Estas API representan interfaces que se diseñaron hace años, muchas de las cuales ya no funcionan. El equipo está en proceso de quitar las referencias a estos puntos de conexión en la próxima versión del SDK del último trimestre de 2020. A continuación se muestran las API que se van a retirar. Los partners deben asegurarse de no hacer a estos URI a partir de entonces.

**Solicitudes de servicio de partners**: estos puntos de conexión permitían a los partners obtener cadenas de temas de soporte técnico y enviar solicitudes de servicio de partners mediante programación. Con el tiempo, esta API se volvió inestable y dejó de admitirse. Los partners se dirigen a la interfaz de usuario del Centro de partners para crear vales de soporte técnico de partners. La experiencia del usuario ofrece información adicional a los partners durante la creación de casos de soporte técnico, como los pasos recomendados y los documentos del área con la que tienen problemas. El equipo del Centro de partners también ha mejorado la experiencia de creación de vales de partners mediante la personalización de los formularios de solicitud de servicio para capturar información específica del área problemática. La información recopilada por los formularios en línea ayuda a los ingenieros de soporte técnico a resolver los problemas de forma más rápida y precisa. No se debe hacer referencia a estos URI ni depender de estos. Se retirarán a finales de agosto de 2020. Tenga en cuenta que estos cambios solo afectan a los temas de obtención de soporte técnico y a los métodos de creación de solicitudes de servicio de **partners**. Todas las demás API relacionadas con el soporte técnico deberían seguir funcionando según lo previsto.

- {base-URL}/v1/servicerequests/supporttopics
- {base-URL}/v1/servicerequests/{agent-locale}

**Incidentes de servicios de Office**: estos puntos de conexión se diseñaron para permitir que a los partners obtener una vista agregada de incidentes o interrupciones de servicio en todos los servicios de los partners. Sin embargo, los datos devueltos por estos puntos de conexión han quedado obsoletos y no son útiles. Estos puntos de conexión se retirarán a finales de julio. En adelante, el equipo del Centro de partners volverá a evaluar los escenarios de mantenimiento e incidentes de servicio de todos los clientes en busca de una oportunidad de agregar valor en esta área. Se deben quitar las referencias a estos URI.

- {base-URL}/v1/serviceincidents/Office/Count
- {base-URL}/v1/serviceincidents/office

Los partners pueden usar la [API de comunicaciones de servicio de Office 365](https://docs.microsoft.com/office/office-365-management-api/office-365-service-communications-api-reference) para obtener el estado actualizado de los servicios de atención al cliente.

**TopProduct**: este punto de conexión se creó como una prueba de concepto, que nunca estuvo destinada a la disponibilidad general. Se diseñó para que algún día los partners pudieran administrar las ofertas principales que se representarían en el catálogo del Centro de partners. Sin embargo, esta API nunca se desarrolló completamente ni se terminó, y se deshabilitará el 31 de julio de 2020. Los partners deben quitar las referencias a esta.

- {base-URL}/v1/preferences/catalog/products

## <a name="next-steps"></a>Pasos a seguir

- Los partners deben asegurarse de que sus soluciones ya no hacen referencia a las API anteriores.
- Los partners del programa Proveedor de soluciones en la nube pueden visitar el [SDK del Centro de partners y la comunidad de API Yammer](https://www.yammer.com/cloudpartnercommunity/#/Threads/show?threadId=786951767490560) para formular preguntas si necesitan más aclaraciones.

## <a name="sdk-release-on-net-standard-v1161-preview"></a><a name="8"></a>Versión del SDK en .NET Standard (v1.16.1-preview)

### <a name="categories"></a>Categories

- Fecha: 21-07-2020
- Funcionalidades

### <a name="impacted-audience"></a>Audiencia afectada

Partners de factura directa y proveedores indirectos que participan en el programa CSP y que usan el SDK de .NET del Centro de partners. 

### <a name="summary"></a>Resumen

A partir del 21 de julio de 2020, los partners pueden empezar a descargar la versión preliminar del **SDK de .NET Standard del Centro de partners de Microsoft**, SDK de .NET v1.16.1-preview, junto con los **ejemplos actualizados públicos del SDK del Centro de partners de GitHub**.

### <a name="details"></a>Detalles

Los partners de facturación directa y los proveedores indirectos que participan en el programa CSP pueden descargar la versión preliminar más reciente del SDK de .NET Standard. El SDK del Centro de partners de Microsoft se migrará de .NET Framework a la plataforma .NET Standard 2.0. Esto hará que el SDK sea compatible con la aplicación existente mediante el .NET Framework 4.6.1 y versiones posteriores. La plataforma .NET Standard será compatible con .NET Core 2.0 y versiones posteriores.

Una gran cantidad de partners ha solicitado el SDK de .NET Standard. La versión actual del SDK obliga a los partners a usar aplicaciones escritas en ASP.NET Framework, que es una solución lenta y costosa en comparación con las aplicaciones de .NET Core. Por lo tanto, la migración a .NET Standard puede ayudar a resolver los problemas de rendimiento o compatibilidad con .NET Core.
 
**Aspectos destacados**
- No hay más versiones de Microsoft para .NET Framework 4.8, ya que es la última de la línea.
- Limita el desarrollo futuro, ya que la mayoría de las bibliotecas se publicarán para .NET Core o .NET Standard, que podrían no ser compatibles con la versión del marco actual.
- .NET Core es conveniente para su uso con contenedores debido a su naturaleza ligera.
- El entorno de tiempo de ejecución de alto rendimiento del servidor para Windows Server y Linux convierte a .NET Core en un marco web de rendimiento superior.

### <a name="next-steps"></a>Pasos a seguir

- Descargar la última versión del [SDK de .NET v1.16.1-preview](https://www.nuget.org/packages/Microsoft.Store.PartnerCenter/1.16.1-preview)
- Versión: SDK de .NET v1.16.1-preview 
- Código de ejemplo: [Código de ejemplo](https://github.com/microsoft/Partner-Center-DotNet-Samples)
- Versión preliminar disponible el: 21/7/2020 
- GA: 3 semanas desde la versión preliminar (para dejar tiempo suficiente para la migración a los partners) 
- No existen nuevos cambios en la API en la versión actual

## <a name="cloud-solution-provider-csp-incentive-for-eligible-microsoft-commercial-marketplace-offers-enhanced-for-fy21"></a><a name="7"></a>Incentivo del Proveedor de soluciones en la nube (CSP) para las ofertas elegibles y mejoradas de marketplace comercial de Microsoft para el año 2021

### <a name="categories"></a>Categories

- Fecha: 22-07-2020
- Funcionalidades

### <a name="impacted-audience"></a>Audiencia afectada

Partners de factura directa y proveedores indirectos que participan en el programa CSP

### <a name="summary"></a>Resumen

A partir del 1 de julio de 2020, los partners de factura directa y los proveedores indirectos del programa CSP pueden ganar incentivos en las transacciones de ofertas de infraestructura como servicio (IaaS) válidas.

### <a name="details"></a>Detalles

Los partners de factura directa y los proveedores indirectos que participan en el programa CSP pueden realizar transacciones de la selección de ofertas de terceros para sus clientes en [marketplace comercial de Microsoft](https://azure.microsoft.com/overview/commercial-marketplace). Como recompensa por el aumento de los ingresos de marketplace comercial, los partners de factura directa y los proveedores indirectos del programa CSP aptos ganarán un incentivo en las transacciones de ofertas válidas.

A partir del 1 de julio de 2020, puede ganar un incentivo en las transacciones de ofertas de IaaS válidas. El incentivo para las ofertas de software como servicio (SaaS) válidas continuará durante el año fiscal 2021.

Puede encontrar la lista más reciente de ofertas de SaaS e IaaS válidas para incentivos en el [sitio web de partners de Microsoft](https://aka.ms/partnerincentives). La lista de ofertas válidas se actualiza mensualmente y se publica en las recopilaciones de recursos del programa de incentivos CSP Direct Bill Partner y el programa de incentivos CSP Indirect Provider. Los partners del programa CSP pueden iniciar estas transacciones a través del Centro de partners o de Azure Portal.

Para ganar incentivos, deberá cumplir todos los requisitos de elegibilidad e inscripción, tal y como se describe en la guía del programa de incentivos pertinente. Las guías de programas y otros recursos útiles de incentivos están disponibles en el [sitio web de partners](https://aka.ms/partnerincentives).

### <a name="next-steps"></a>Pasos siguientes

- Vea el anuncio completo y obtenga más información sobre marketplace comercial en la colección de la galería de recursos de preparación de operaciones: [Incentivo del programa CSP para las ofertas elegibles de marketplace comercial de Microsoft](https://partner.microsoft.com/resources/collection/csp-incentive-for-microsoft-commercial-marketplace-offers#/).
 
- Revise la lista de las ofertas de SaaS e IaaS válidas al principio de cada mes.
 
- Atraer a los clientes para que ofrezcan estos servicios como parte de la relación de facturación del programa CSP.

## <a name="new-capabilities-added-to-the-insights-reports-in-partner-center"></a><a name="6"></a>Nuevas funcionalidades agregadas a los informes de conclusiones del Centro de partners 


### <a name="categories"></a>Categories

- Fecha: 16-07-2020
- Funcionalidades
 
 
### <a name="impacted-audience"></a>Audiencia afectada

Todas las organizaciones asociadas de Microsoft Partner Network 

### <a name="summary"></a>Resumen

Se han agregado nuevas funcionalidades a los informes de conclusiones del Centro de partners, lo que permite a los asociados realizar análisis de datos ad hoc en función de sus necesidades empresariales y ver una vista holística del entrenamiento realizado por los empleados de la organización.  Estas nuevas funcionalidades aumentan el sólido conjunto de [informes de conclusiones existentes](../partner-center-insights.md).

Información general de las nuevas funcionalidades:

- **Exportación de datos sin procesar** habilitada para los detalles del cliente, los detalles de la suscripción, el uso de Microsoft Azure, el uso de Office 365, el uso de Dynamics 365, el uso de Enterprise Mobility + Security (EMS), el uso de Power BI, el entrenamiento y las certificaciones, y Microsoft Learn. Para obtener más información, consulta [este artículo](../pci-download-reports.md).

- **Nuevo panel de entrenamiento** que proporciona una vista holística del entrenamiento realizado por los empleados de la organización. Para obtener más información, consulta [este artículo](../pci-training-dashboard.md).

### <a name="what-is-insights"></a>¿Qué es Conclusiones?

El panel Conclusiones del Centro de partners proporciona una vista unificada de 360 grados de los indicadores clave de rendimiento (KPI) de su organización de partner a través de productos en la nube, como Office 365, Azure, Dynamics 365 y modelos de licencia como Proveedor de soluciones en la nube (CSP) y Contrato Enterprise (EA). Intelligent Insights se proporciona para que entienda claramente su negocio con Microsoft y pueda tomar decisiones basadas en datos con confianza.   

Para ver este panel, seleccione **Conclusiones** en el menú de navegación izquierdo del Centro de partners. Este panel usa los mismos datos subyacentes que la herramienta de generación de informes MyInsights actual, pero ofrece objetos visuales más completos y más funcionalidades de informes perspicaces e indicadores de rendimiento clave que abarcan los clientes, las suscripciones, el uso de licencias, las competencias y las ventajas. Vuelva a consultarlo con frecuencia, ya que seguiremos incorporando nuevas funcionalidades e informes al panel Conclusiones.    

### <a name="next-steps"></a>Pasos siguientes

Le recomendamos que explore estas nuevas funcionalidades y comparta sus comentarios con las herramientas de comentarios en línea disponibles en la pestaña **Conclusiones**.

## <a name="perpetual-software-in-the-cloud-solution-provider-program"></a><a name="5"></a>Software perpetuo del programa Proveedor de soluciones en la nube 


### <a name="categories"></a>Categories

- Fecha: 15-07-2020
- Ofertas/mercados
- Estado: disponible para el conjunto de proveedores indirectos cualificados y sus revendedores indirectos. Disponible para el público general en enero de 2021.
 
 
### <a name="impacted-audience"></a>Audiencia afectada

Distribuidores indirectos y proveedores indirectos cualificados del programa CSP 

### <a name="details"></a>Detalles

En Microsoft, nos comprometemos a ofrecer soporte técnico a todas las organizaciones en su proceso de transformación digital. Para ello, invertimos no solo en tecnología, sino también con el examen del recorrido de un extremo a otro de clientes y partners, desde la adquisición de la tecnología hasta la experiencia de soporte técnico. A través de este trabajo hemos identificado una oportunidad para transformarnos mediante la inversión en nuestra plataforma de comercio.  También estamos simplificando los programas de licencia a la vez que desarrollamos la nueva experiencia de comercio para poder mejorar la forma en que proporcionamos más oportunidades a nuestros partners. 

A medida que mejoramos las funcionalidades de comercio para ofrecer a clientes y partners una experiencia de compra mejor, expandimos la cartera de ofertas para incluir la disponibilidad de las licencias de software perpetuas en el programa Proveedor de soluciones en la nube (CSP). La adición de licencias de software al programa CSP permite a los partners participantes centrarse en ofrecer diversas soluciones y servicios administrados a sus clientes. También permite la transición de los clientes a la nube y ofrece a los partners la flexibilidad necesaria para los entornos de nube híbrida.   

A partir del 1 de julio, se reforzará esta capacidad a través de un conjunto de proveedores indirectos cualificados y de sus revendedores indirectos, que podrán gestionar licencias de software local con los clientes del programa CSP. Microsoft administra los partners cualificados, que deben ser proveedores indirectos del programa CSP activos y distribuidores autorizados de Microsoft para que el programa de licencias abiertas opere en una geografía actualmente admitida para la venta de software en el programa CSP. A partir de enero de 2021, Microsoft escalará aún más esta capacidad y permitirá a todos los partners de facturación directa, proveedores indirectos y distribuidores indirectos del programa CSP vender licencias de software en el programa CSP.    

### <a name="next-steps"></a>Pasos siguientes

- Revise todos los materiales de preparación de la colección de la galería de recursos de preparación de operaciones:  [Software perpetuo del programa Proveedor de soluciones en la nube](https://partner.microsoft.com/resources/collection/software-in-csp#/).
